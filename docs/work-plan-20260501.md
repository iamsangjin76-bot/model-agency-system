## 작업 1. Rate Limiting 🔴

**문제:** 로그인 API에 시도 횟수 제한 없음 → 브루트포스 공격 취약  
**파일:** `backend/requirements.txt`, `backend/app/main.py`

**세부 단계:**
1. `requirements.txt`에 `slowapi>=0.1.9` 추가
2. `main.py`에 Limiter 초기화 및 미들웨어 등록
3. `/api/auth/login` — `10/minute` 제한
4. `/api/auth/token` — `10/minute` 제한
5. GitHub 푸시 → Railway 자동 배포
6. 검증: 로그인 11회 연속 시도 → 429 Too Many Requests

## 작업 2. CORS 프로덕션 설정 🟡

**문제:** `localhost`, `192.168.x.x` 주소가 운영 서버 CORS에 포함  
**파일:** `backend/app/config.py`

**현재 상태:**
```python
CORS_ORIGINS: list = [
    "http://localhost:5173",   # ← 제거 필요
    "http://localhost:5174",   # ← 제거 필요
    "http://127.0.0.1:5173",   # ← 제거 필요
    ...
]
```

**세부 단계:**
1. `CORS_ORIGINS` 환경변수로 외부 관리 가능하도록 변경
2. 기본값을 `https://project-m.co.kr` 만 허용
3. 로컬 개발 시 `.env`에서 localhost 추가 가능하도록 유지
4. GitHub 푸시 → Railway 자동 배포

## 작업 3. DEBUG=False 🟡

**문제:** 기본값 `DEBUG=True` → 운영 서버에서 오류 내부 정보 노출  
**방법:** Railway Variables 탭에서 수정

**세부 단계:**
1. Railway 대시보드 → Variables 탭
2. `DEBUG` 값 → `False` 로 변경
3. Deploy 클릭

## 🎓 이번 세션에서 얻은 교훈

### 1. 중첩 라우트의 catch-all은 각 레벨에 필요 🆕
- App.tsx의 `path="*"`는 `/dashboard/*` 내부 경로에 도달 불가
- DashboardPage 내부 nested `<Routes>`에도 별도 `path="*"` 필요
- **교훈**: 중첩 라우트 구조에서는 각 `<Routes>` 레벨마다 catch-all 확인

### 2. alert() → toast 교체는 기계적 but 안전하게 🆕
- 9개 파일 16건의 alert()를 toast로 교체
- window.confirm()은 교체 범위 외 — 삭제 확인은 모달 다이얼로그가 적절
- **교훈**: UI 피드백 수단 교체 시 비즈니스 로직 변경 없이 알림 수단만 교체하는 원칙

### 3. 공용 컴포넌트 추출은 복붙 제거의 핵심 🆕
- Spinner 28줄 1개 컴포넌트로 15+곳의 복붙 코드 제거
- **교훈**: 동일 패턴이 3곳 이상 반복되면 공용 컴포넌트 추출 후보

### 4. 검색/크롤링 기능은 구현계획서에 누락되어 있었음 🆕
- 페이지 UI(NewsSearchPage, ImageSearchPage, SNSAnalysisPage)는 있으나 실제 동작 없음
- 구현계획서 작성 시 "UI 존재 = 기능 완성" 착각 주의
- **교훈**: 빈 껍데기 페이지도 로드맵에 명시적으로 포함해야 함

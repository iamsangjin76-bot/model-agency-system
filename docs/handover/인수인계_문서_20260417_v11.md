- **이 원칙은 기억 시스템에 추가 완료**

### 2. 캐스팅 필드명 불일치 — 프론트↔백엔드 인터페이스 검증 필수 🆕

- 프론트엔드가 `project_name`을 보내고 백엔드가 `title`을 기대하는 불일치 → 422 에러
- 새 도메인 작업 시 Phase 1로 필드명 정렬을 반드시 포함할 것

### 3. DB 마이그레이션 선제 처리 🆕

- `is_active` 컬럼 추가 시 ALTER TABLE을 코드 수정과 동시에 실행해야 함
- SQLAlchemy `checkfirst=True`는 테이블 생성만 담당, 기존 테이블 컬럼 추가는 안 됨
- F-2에서 실수 → F-3에서 선제 해결 (SPEC 리스크에 명시)

### 4. `[object Object]` 에러 패턴 🆕

- FastAPI 422 응답의 `detail`이 배열일 때 `new Error(배열)` → `[object Object]`
- `auth-api.ts`의 에러 핸들러에서 `Array.isArray` 처리 추가로 해결

### 5. 대규모 파일 리팩토링 시 컴포넌트 추출 패턴 🆕

- CastingPage 479→211줄, SettlementPage 657→158줄

## 🎓 이번 세션에서 얻은 교훈

### 1. reviewer의 curl 테스트만으로는 프론트엔드 검증 불가 🆕
- curl로 API 200 응답 확인 → "통과" 판정 → 실제 브라우저에서 안 됨
- **원인**: UI disabled 조건, navigate 경로, 드롭다운 상태 등은 curl로 확인 불가
- **교훈**: reviewer 프롬프트에 "프론트엔드 코드를 직접 읽고 isEdit/disabled 조건 확인" 의무화
- 최종 검증은 반드시 사용자 브라우저 확인

### 2. exclude_admin_id로 자기 알림 제외 = 1인 테스트 시 혼란 🆕
- admin으로 모델 등록 → admin에게는 알림 안 옴 (설계대로)
- 다른 계정(manager1)으로 로그인해야 알림 확인 가능
- **교훈**: 알림 테스트 시 2개 이상 계정 필요

### 3. optimistic update 패턴이 알림 UX에 필수 🆕
- await API 후 state 변경 → 느리거나 실패하면 UI 무반응
- state 먼저 변경 + API는 background → 즉각적 UX
- catch에서 rollback하면 더 정확하지만, 알림 읽음은 non-critical이므로 생략 가능

### 4. Tailwind xl: breakpoint 활용 부재 🆕
- 기존 코드가 lg:(1024px)에 모든 레이아웃 전환을 몰아넣음
- 사이드바 + 테이블 + 폼이 동시에 1024px에서 전환 → 과밀
- **해결**: xl:(1280px)에 폼 4컬럼 복원, 사이드바 w-72 확장 등 분산

### 5. 대시보드 통계 카드 `<div>` → `<Link>` 패턴 🆕
- hover 스타일만 있고 실제 클릭 동작 없는 "가짜 버튼" 주의
- 클릭 가능해 보이는 요소는 반드시 Link/button이어야 함

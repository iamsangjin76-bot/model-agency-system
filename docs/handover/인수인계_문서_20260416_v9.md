## 🗺️ Phase D-3 — 히스토리 정리 (선택)

### 개요

Phase D-2에서 추적 파일은 정리했지만, **git history에는 삭제된 파일들의 blob이 여전히 남아있음**. `git clone` 시 전체 히스토리를 받으므로 용량 영향 있음.

### 도구

- `git filter-repo` (권장) — Python 기반, `git filter-branch`보다 안전·빠름
- 주의: **히스토리 rewrite = force push 필요 = 비가역적**

### 판단 기준

| 상황 | 권장 |
|---|---|
| 1인 개발 + private repo | filter-repo 해도 됨 |
| 팀원 합류 예정 | 합류 전에 하거나 안 함 |
| 지금 clone 속도 문제 없음 | 급하지 않음, 보류 가능 |

### 우선순위

**낮음 (P3)**. 추적 파일 정리(Phase D-2)가 실질적 효과의 대부분. 히스토리 정리는 편의성 개선 수준.

## 2. Claude Code CLI 묶음 자동화 대기 상태 — 핵심

### 2.1 대기 모드 트리거

기존 Claude Code CLI 창이 살아있다면, 다음 메시지 한 줄이 트리거:

```
J-8b 시각 검증 완료
```

이 메시지를 받으면 Claude Code가 자동으로 다음 3단계를 순차 진행:

1. `git push origin master` (J-8b commit `f9c8feea` push)
2. `/auto fix "DashboardPage.tsx:278 PlaceholderPage 다크 짝 1줄 보정"` (J-8d, Simple)
3. `/auto plan "J-8c ImageSearchPage 자동 매칭 패턴 NewsSearchPage 복제" --skip-prd` → `/auto go SPEC-{ID}` → sync 직전 일시정지 (J-8c, Medium)

### 2.2 만약 Claude Code CLI 창이 종료됐다면

대기 모드는 **휘발성**이라 종료 시 사라집니다. 새 세션에서는 묶음 명령문을 다시 붙여넣어야 합니다.

**묶음 명령문 원본** (재사용용):

```
J-8 잔여 작업 자동화 묶음 진행해주세요. 분기점에서만 저에게 보고하고 나머지는 알아서 진행합니다.

[작업 1] J-8b 시각 검증 후 push
- 제가 별도 메시지로 "J-8b 시각 검증 완료"라고 보고할 때까지 대기
- 보고받으면 즉시: git push origin master
- push 결과(원격 master 해시) 보고

[작업 2] J-8d 다크모드 1줄 보정 (Simple)

[전체 원칙]
- planner 권고가 모호하거나 architect 필요 판단 시 즉시 일시정지하고 보고

준비됐으면 "묶음 작업 대기 중, J-8b 시각 검증 보고 기다림" 한 줄로만 답하고 대기 모드 진입해주세요.
```

## 11. 메모 — 새 Claude에게

이번 세션은 비교적 단순한 환경 점검 + sync 작업이었지만, 시각 검증을 우회하고 sync로 직행한 메타 결함이 발생했습니다. 새 Claude는 다음을 명심해주세요:

**1. 시각 검증은 협상 불가**
- v21 §3.1 흐름은 "5173 재기동 → 시각 검증 → commit + sync"입니다
- 코드가 디스크에 있다는 사실만으로 "구현 완료"로 판단하면 안 됩니다
- Jin님이 "잘 됨"이라고 보고하기 전까지는 sync 마킹도, push도 금지

**2. 모델 선택 + 한 단계 제공이 첫 응답 형식**
- Jin님이 "이어서 작업"이라고 하시면 즉시 인수인계 검색 → 보고
- 보고 후 다음 한 단계만 명령 포함 안내 (두 단계 동시 금지)
- 모델 선택은 응답 중간 표가 아니라 명령 직전 눈에 띄는 위치에

**3. PowerShell vs CMD 구분 필수**
- Jin님은 PowerShell + CMD를 섞어 사용하십니다
- `cd /d` 같은 CMD 명령을 PowerShell에 안내하면 즉시 거부됩니다
- 어느 쉘인지 모를 때는 먼저 물어보세요

**4. Poetry 미사용**
- 백엔드는 venv 기반입니다. `poetry run` 명령 안내 금지
- 정확한 명령은 §3.2 참조

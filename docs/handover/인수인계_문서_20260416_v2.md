## ⚠️ 주의사항 / 알려진 이슈

1. **GitHub 히스토리에 과거 민감 파일 잔존**
   - `.env` (플레이스홀더), `venv/`, `.db` 가 initial commit 에 포함
   - Private repo + 플레이스홀더였기에 실질 위험은 낮음
   - 완벽 정리 원하면 `BFG Repo-Cleaner` 로 히스토리 재작성 가능 (지금은 비권장)

2. **대시보드 "데이터 없음" 표시는 정상 동작**
   - DB 에 실제 데이터가 없어서 차트가 빈 상태
   - 버그 아님. 해결 = 시드 데이터 투입 (우선순위 7)

3. **"최근 활동" 패널은 하드코딩 더미 데이터**
   - "김모델 / 홍길동 / 박지민 / 이수진" 은 화면 확인용 예시
   - 실제 DB 로 연결 필요 (우선순위 8)

4. **`requirements.txt` 에 `httpx` 누락**
   - 로컬에서는 수동 `pip install httpx` 했음
   - 새 환경 세팅 시 문제 생길 수 있음 → 우선순위 9 로 추가

5. **Claude Code `PostToolUse:Bash hook error`** 🆕
   - 현상: 명령 실행 후 가끔 `"failed to list CI runs: exit status 4"` 메시지 출력됨
   - 영향: **없음** (GitHub Actions 상태 조회 훅이 실패한 것일 뿐, 실제 명령 결과는 정상)
   - 대응: 무시해도 됨. 거슬리면 훅 설정에서 비활성화 가능.

## 🎓 이번 세션에서 배운 교훈

1. **`.gitignore` 변경 시 체크리스트 필수**
   - 추적 해제(`git rm --cached`)와 `.gitignore` 항목 추가는 **한 쌍**으로 움직여야 함
   - 한쪽만 하면 추적은 안 되지만 Untracked 노이즈가 계속 발생
   - 이후 `.gitignore` 건드릴 때는 "추적 해제 + ignore 추가 + git status 검증" 3단 체크

2. **세션 간 상태 차이 감지는 아키텍트의 역할**
   - 비개발자는 각 세션에서 일어난 일을 일일이 기억하기 어려움
   - 새 세션 시작 시 **"인수인계 문서 vs 실제 저장소 상태"** 일치 여부를 아키텍트(Opus 4.6)가 먼저 점검해야 함
   - 이번 세션 초반에 문서-사용자 말 불일치를 짚은 것이 좋은 패턴

3. **커밋 메시지는 "English summary + 한국어 본문" 규칙 정착 중**
   - 향후 커밋 히스토리 가독성을 위해 계속 이 패턴 유지

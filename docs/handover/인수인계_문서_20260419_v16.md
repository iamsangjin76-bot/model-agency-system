## 🎓 이번 세션에서 얻은 교훈

### 1. 모델 선택 드롭다운 vs 자동 매칭 🆕
- 개발자 관점의 "정확한 매칭" (드롭다운) vs 사용자 관점의 "자연스러운 흐름" (자동 매칭)
- **교훈**: 사용자가 쓰기 편하게 만드는 게 개발의 목적. 네이버 검색하듯 자연어 입력→자동 처리가 정답.

### 2. 미등록 모델 자동 생성 패턴 🆕
- 검색어에서 모델 매칭 실패 시 → 모델명 입력 모달 → modelsAPI.create() → 저장
- modelsAPI.create()의 필수 필드(name + model_type) 확인 필수 — 422 에러 발생했었음

### 3. 구글 API 설정 복잡성 🆕
- Google Cloud Console 프로젝트 선택, Custom Search API 활성화, CX(검색엔진 ID) 생성, API 키 프로젝트 일치 — 4단계 모두 맞아야 동작
- **교훈**: 외부 API 연동 시 사용자가 설정할 수 있도록 상세 가이드 필요

### 4. reviewer + security-auditor 빠뜨리지 말 것 🆕
- Complex 워크플로에서 executor 끝나고 바로 push하려 했다가 Jin님이 지적
- **교훈**: WORKFLOW 규칙(executor → reviewer → security-auditor) 절대 생략 금지

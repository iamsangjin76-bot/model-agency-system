## 5. 결정 보류 / 잠재 이슈

### 5.1 search.pstatic.net 도메인 화이트리스트 누락 가능성

J-8a `IMAGE_PROXY_ALLOWED_DOMAINS` 9개:
```
imgnews.naver.net, naver.net, blogfiles.naver.net, postfiles.pstatic.net,
ssl.pstatic.net, i1~i3.ruliweb.com, image.fmkorea.com, img.insight.co.kr
```

그러나 실제 검색 API thumbnail_url은 `https://search.pstatic.net/...` 도메인.
- `search.pstatic.net`은 **9개 화이트리스트에 없음**
- 그러나 `validate_proxy_host` suffix 매칭 룰: `search.pstatic.net`은 `naver.net` suffix 미매칭, `pstatic.net` suffix도 9개에 미포함
- `postfiles.pstatic.net` / `ssl.pstatic.net`만 등록됨 → search.pstatic.net 차단 가능성 높음

**확인 방법**: V1 실행 시 Network 탭 응답 코드 → 403이면 도메인 추가 필요. 추가 도메인 후보:
- `search.pstatic.net` (썸네일 CDN)
- `pstatic.net` (와일드카드 suffix — 모든 pstatic.net 서브도메인 허용)

이 경우 `.env` `IMAGE_PROXY_ALLOWED_DOMAINS`에 추가 + 백엔드 재시작 필요.

### 5.2 ModelImageGallery proxify wrap의 의미

`imageUrl()` 헬퍼는 `/uploads/...` server-relative 반환 → `proxify()` 자동 우회 (no-op). 적용한 가치:
- onError 핸들러 (로컬 파일 누락 시 placeholder)
- 미래 안전망 (localPath가 외부 URL이 될 경우)

V8 검증 시 Network 탭에서 `/api/proxy/image` 호출 0건 확인 필수 (호출되면 정책 위반).

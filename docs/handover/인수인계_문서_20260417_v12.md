## 🎓 이번 세션에서 얻은 교훈

### 1. schemas/ 패키지가 schemas.py를 가림 🆕
- `backend/app/schemas/` 디렉토리가 존재하면 `backend/app/schemas.py` 파일은 무시됨
- **실제 사용되는 파일**: `schemas/agency.py`의 ModelCreate, ModelUpdate
- schemas.py에 필드를 추가해도 반영 안 됨 → 디버깅 시간 낭비 원인

### 2. vite.config.ts 프록시 누락 = 브라우저에서 이미지 404 🆕
- `/uploads` 경로가 Vite dev proxy에 없으면 브라우저에서 이미지 로드 불가
- curl은 백엔드 직접 호출이라 성공 → 프론트엔드에서만 실패하는 함정

### 3. SQLAlchemy Enum: name vs value 혼동 🆕
- SQLAlchemy는 Python enum의 **name** (대문자: 'MALE', 'FEMALE')을 DB에 저장
- Pydantic str-enum의 **value** (소문자: 'male', 'female')와 불일치
- `setattr(model, 'gender', pydantic_enum)` → DB에 소문자 저장 → ORM 읽기 시 LookupError
- 해결: `Gender(pydantic_enum.value)` 로 SQLAlchemy enum 멤버로 변환

### 4. "프론트엔드 수정 불필요" 판단 금지 🆕
- curl 테스트만 통과시키고 "프론트엔드는 이미 완성" 으로 단정하면 안 됨
- 반드시 프론트엔드 코드(handleSubmit, API 호출, state 관리)를 직접 읽고 검증해야 함

### 5. 부분 수정 → 확인 → 또 수정 반복 금지 🆕
- 한 기능의 전체 파이프라인(백엔드 → API → 프론트엔드 → 브라우저)을 한번에 점검
- 사용자에게 "확인해주세요" → "안 돼요" → "다시 고칠게요" 반복은 신뢰 저하

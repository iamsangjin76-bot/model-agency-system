# 마지막 작업 검토 보고서

## Context (작업 배경)

검토 대상: 커밋 `be95dad7` — *feat(export,pdf,pptx): A4 landscape profile export with 4 templates* (2026-04-28).

이 커밋은 4개 템플릿(신인A/B, 인플루언서, 외국인모델)으로 A4 가로 프로필을 PDF/PPT로 내보내는 기능을 추가했습니다. 1,515줄 추가 / 214줄 삭제 규모이며, 프론트엔드 인쇄 파이프라인(`printProfile.ts`), 페이지(`ProfileExportPage.tsx`), 백엔드 PPTX 생성기(`pptx_service.py`), 라우터(`export.py`), 도메인 인터페이스(`domain-api.ts`)가 동시에 수정되었습니다.

작업은 동작하지만, 프로젝트의 **CLAUDE.md 규칙 위반** 및 **보안·정합성 결함**이 발견되어 후속 정리가 필요합니다.

## 핵심 이슈 요약 (Jin님 비개발자용 정리)

### 🔴 즉시 조치 필요

1. **파일 길이 규칙 위반 (CLAUDE.md: 300줄 한도)**
   - `frontend/src/utils/printProfile.ts` — **581줄** (281줄 초과)
   - `frontend/src/pages/ProfileExportPage.tsx` — **472줄** (172줄 초과)
   - `backend/app/services/pptx_service.py` — **384줄** (84줄 초과)
   - → 한 파일이 너무 많은 일을 하고 있어, 나중에 수정할 때 사고가 날 확률이 높습니다.

2. **보안 취약점 (경로 탈출 가능)** — `backend/app/services/pptx_service.py:124-135`
   - `_resolve_path()` 함수가 사용자 입력 URL을 충분히 검증하지 않아, 악의적 요청 시 서버의 다른 파일(`/etc/passwd` 등)을 PPT에 끼워 넣을 수 있는 위험.
   - → `os.path.abspath()` 정규화 + `upload_dir` 하위 강제 검증 필요.

3. **데이터 스키마 불일치 (앨범·뮤직비디오 경력 누락)**
   - 프론트(`domain-api.ts`)와 인쇄(`printProfile.ts`)는 `career_album`, `career_music_video` 필드를 사용.
   - 그러나 백엔드 `pptx_service.py`의 `CAREER_FIELDS`(L47-52)에는 누락 → **PPT 다운로드 시 앨범/뮤직비디오 경력이 빈칸**.

### 🟡 품질 개선 권장

4. **중복 정의 3곳** — 색상 팔레트(`PALETTE`), 모델 타입 라벨, 성별 라벨이 `printProfile.ts`, `pptx_service.py`, `ProfileExportPage.tsx` 세 곳에 따로 적혀 있음. 한 곳만 바꾸면 다른 곳과 어긋남.

5. **이미지 base64 변환 UX 저하** — `printProfile.ts:110-143`. 모델 10명 선택 시 팝업이 열린 뒤 10~20초 대기. 캐싱 또는 진행 표시 필요.

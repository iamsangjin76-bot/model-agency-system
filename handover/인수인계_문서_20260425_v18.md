## 🔥 핵심 발견 4건 (인수인계 가치 큼)

### 1. `auto canary` 명령 자체가 Autopus CLI(v0.39.1)에 미구현

`auto --help` 36개 서브커맨드 목록(agent / arch / check / completion / config / connect / desktop / docs / doctor / effort / experiment / hash / help / init / issue / learn / lore / lsp / mcp / orchestra / permission / pipeline / platform / react / search / setup / skill / spec / status / telemetry / terminal / test / update / verify / version / worker) 중 **canary 부재**. v17 인수인계의 17개 서브커맨드 분류와 실제 CLI 사양 불일치. canary.md 사용 예 섹션에 ⚠️ 표시됨. 향후 SPEC-CANARY-001 별도 도입 또는 수동 검증으로 대체.

### 2. Node.js v24.14.1은 비-LTS

운영 환경의 Node.js는 v24.14.1이지만 v24는 **LTS 아님** (현재 LTS는 v22 Iron). `package.json`의 `engines` 필드 + `.nvmrc` 모두 부재로 버전 강제 없음. Phase I-1 Electron 패키징 시 LTS 버전(v22)으로 고정 권장.

### 3. 시드 데이터와 enum 정의 불일치

- **ClientGrade**: enum 4등급(VIP/Gold/Silver/Normal) vs seed.py 3등급만 시드 (Silver 부재)
- **ContractType**: enum 4종(EXCLUSIVE/PROJECT/ANNUAL/EVENT) vs seed.py 2종만 시드 (PROJECT/ANNUAL 부재)

향후 Silver 클라이언트 또는 PROJECT/ANNUAL 계약 추가 시 시드 보강 검토.

### 4. KPI 카드 "공유 링크" vs Share 기능 부재 모순 가능성

`DashboardPage.tsx` 149줄에 KPI 카드 "공유 링크" 정의됨 (path: `/dashboard/share`). 그러나 v17 알려진 이슈 #2 "Share 링크 기능 없음"(옵션 B 후 잃은 기능)과 모순. App.tsx 라우트(436줄)는 `<Route path="share" element={<PlaceholderPage title="외부 공유" />} />` — **PlaceholderPage(껍데기)**. 카드만 있고 실 기능 부재. J-6 검증 시 명확화 또는 카드 제거 검토 권장.
